PiSiDo is a simple PISI package creation software. It collects some information about source code, package, compilation process and packager and creates proper files to make a .pisi package.


First, you select a work directory to define where this pisi build information will save. Then define source code archive, and finally compilation python script to build the code. Here is how the PiSiDo work directory created:

  • PiSiDo Work Directory (see Package section)

    • <source_folder>.tar.gz (optional)

    • <paket_name>-<version>-<others>.pisi (optional)

    • <package_name> (directory)

      • pspec.xml

      • action.py

      • files (directory) (optional)

        • <package_name>.desktop (optional)

        • <package_name>.png (optional)


There are two buttons and a check box under the window. Clear button resets all fields to initial states. Create button creates PISI build files Build check box is not checked. If checked, a dialog will promp for password and than a console will open and starts building package.


Caution ! PiSiDo will overwrite to the files that selected working dir. (Except .png file under files dir.)


Saving And Loading Package Build Info

You can find an option to save and load package build information in the File menu. You can save and load field values by this option. This operation will not affect anything on the PiSiDo working directory.



Building Tip

First you should create with Create button, and then if you build from Tools menu, pspec.xml file will not overwrite. You can edit pspec.xml file with this way.



Source Code

In this section, we define how to get source code archive.

  • Compressed File : Select a local compressed file. File name will add to archive tag, and sha1sum produced for the archive attribute.

  • Folder Path : Select a local folder. The folder will compress to a file under PiSiDo Work Directory, and behave as Compressed File section above.

  • Url Address : Enter a web site addres that shows exact file location (ex:http://ex.com/file.tar.gz). Full file path will add to archive, and you must fill the SHA1 value for add to the archive attribute.
    This section is usefull for deploying package creation files to other people. They can build from source code with this information. The archive will download from URL and compile.

  • Copy source code to pisi archive directory : If you select this, it will prompt for the password. This item only works with local files. If you want to make package after creating the pisi build files, you should check this. It will copy the archive to pisi work directory for the compilation. This must be done either way.
    If you select URL option, pisi will download the archive to its work dir itself. So, you don't need at URL option.

  • Home Page : This is an information about package. It will show in the package information.

Package

In this section, we define package information and PiSiDo package working directory.

  • PiSiDo Work Dir : This field define the PiSiDo current work directory for the package. All files that created will store in here. You can select a folder with Browse button and open selected folder with Open button.
    These files will create in this folder:

    • pspec.xml : stores information that needed for the package and packaging phases.

    • action.py : a script file that will describe how to compile and install the package.

    • <package_name>.desktop : an icon descriptor for running the application in the package. Created under the “files” folder in work directory. This is optional, see Compilation.

    • <package_name>.png : an icon for visualizing .desktop file. Created under the “files” folder in work directory. This is optional, see Compilation.

  • Name : Defines the package name. This will use in package manager and created pisi file.

  • Version : Defines the package version.

  • Licence : Defines the licence that software in the package uses. This can be more than one. You can edit pspec.xml and add more after creation.

  • IsA : Defines which type of package is (app:gui, app:console, etc.). This can be more than one. You can edit pspec.xml and add more after creation.

  • PartOf : Defines which type of package is (game, hardware, etc.). This can be more than one. You can edit pspec.xml and add more after creation.

  • Summary : This section will show the user in the package manager list.

  • Description : This section will show the user after clicking to package name in the package manager list.

Compilation

In this section, we define compilation process.

  • Select action.py template : When you select a build type on the left, you will get an action.py template on the right side. You can change the python script before the creation.

  • Create .desktop file : If you select this option, you will get a icon and a .desktop file in the files folder. Also, the proper tags in the pspec.xml will add and you can see an icon in the K menu after install the package.

Packager

  • Name : Your name :)

  • E-Mail : Your email. This information wil add to pspec.xml file.
